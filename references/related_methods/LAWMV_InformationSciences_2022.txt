a b s t r a c t
Crowdsourcing provides an efficient way to obtain multiple noisy labels from different
crowd workers for each unlabeled instance. Label integration methods are designed to infer
the unknown true label of each instance from its multiple noisy label set. We argue that
when the label quality is higher than random classification, the more the number of labels,
the better the performance of label integration methods. However, in real-world crowdsourcing scenarios, each instance cannot obtain enough labels for saving costs. To solve this
problem, this paper proposes a novel label integration method called label augmented and
weighted majority voting (LAWMV). At first, LAWMV uses the K-nearest neighbors (KNN)
algorithm to find each instance’s K-nearest neighbors (including itself) and merges their
multiple noisy label sets to obtain its augmented multiple noisy label set. Then, the labels
from different neighbors are weighted by the distances and the label similarities between
each instance and its neighbors. Finally, the integrated label of each instance is inferred by
weighted majority voting (MV). The experimental results on 34 simulated and two realworld crowdsourced datasets show that LAWMV significantly outperforms all the other
state-of-the-art label integration methods.

1. Introduction
Supervised learning [9,37] is a very popular machine learning paradigm and is widely applied in the real world. In supervised learning, large-scale labeled data is essential for improving the learning model’s quality. In recent years, due to artificial
intelligence has developed rapidly, the demand for large-scale labeled data has increased accordingly. However, large-scale
labeled data is hard to obtain. Traditionally, a labeling task is performed by domain experts or well-trained workers. This
ensures the correctness of labels, but it is expensive and time-consuming [13,35,6].
Fortunately, crowdsourcing provides an economical and efficient solution [38]. In crowdsourcing scenarios, the labels are
obtained from crowd workers through online crowdsourcing platforms [1] such as Crowdflower1, Clickworker2 and Amazon
Mechanical Turk (AMT)3. This way can quickly obtain labels for large-scale data at a relatively low cost and has been widely

applied in domains such as natural language processing, image classification and object detection. However, due to the personal
factors of each crowd worker, such as preference, intention and expertise, the label quality is not guaranteed [23,2]. Training
models directly using these noisy labels may cause poor performance. To improve the label quality, researchers have proposed
many effective task assignment strategies [31,10,30], and after task assignment, how to infer the true label of each instance
from its multiple noisy label set obtained from different crowd workers is also a very meaningful topic [22,29,12].
In recent years, label integration for inferring the true label of each instance from its multiple noisy label set has been
extensively studied. Majority voting (MV) is the simplest but often effective label integration method. However, MV naively
assumes that each crowd worker has the same labeling quality. Obviously, this is almost impossible in the real world and
thus hurts the performance of MV. To improve the performance of label integration, researchers have proposed many relatively complex methods from different aspects. For example, [4] proposed the ZenCrowd (ZC) method, which models each
crowd worker’s reliability through an expectation–maximization method. [11] proposed the KOS method, which models
each crowd worker’s reliability through a similar belief propagation method. [14] proposed the Iterative Weighted Majority
Voting (IWMV) method, which uses the integrated labels obtained by MV to estimate each crowd worker’s labeling quality
through an iterative method. [41] proposed the Ground Truth Inference using Clustering (GTIC) method, which uses the features generated from multiple noisy label sets to obtain integrated labels by clustering. Recently, [7] proposed the Multiple
Noisy Label Distribution Propagation (MNLDP) method, which solves the problem of label integration by propagating the
label distributions of instances. Although these methods generally perform better than MV, for better application in the real
world, they still need to be further improved.
In this paper, we argue that when the label quality is higher than random classification, the more the number of labels,
the better the performance of label integration methods [24,7]. However, in real-world crowdsourcing scenarios, each
instance cannot obtain enough labels for saving costs. Therefore, we consider augmenting the multiple noisy label set of each
instance without increasing the cost, and KNN can do exactly that. In some other research fields such as image retrieval
[16,19] and image understanding [17,18], good experimental results have also been achieved by exploring neighbors’ labels.
Therefore, we are more convinced that adapting KNN to crowdsourcing scenarios can solve the problem of insufficient labels
and propose a novel label integration method called label augmented and weighted majority voting (LAWMV). In LAWMV,
we first use the K-nearest neighbors (KNN) algorithm to find each instance’s K-nearest neighbors (including itself) and merge
their multiple noisy label sets to obtain its augmented multiple noisy label set. Then, we weight the labels from different
neighbors by the distances and the label similarities between each instance and its neighbors. Finally, we infer the integrated
label of each instance by weighted MV. To sum up, the main contributions of this paper include:
1. The insufficient label problem for saving cost in the real-world crowdsourcing scenarios is transformed into how to use
existing information to augment labels, which provides a new perspective to solve the label integration problem.
2. A novel label integration method is designed, which uses the KNN algorithm to find each instance’s K-nearest neighbors
(including itself) and merges their multiple noisy label sets to obtain its augmented multiple noisy label set, thereby
weakens the influence of the lower quality labels in the multiple noisy label set.
3. The distances and the label similarities between each instance and its neighbors are used to weight the labels from different neighbors to enhance the impact of neighbors that are more likely to belong to the same class as each instance.
The rest of this paper is organized as follows. Section 2 describes the related work on label integration. Section 3 introduces the proposed LAWMV in detail. Section 4 reports the experiments and results on simulated and real-world datasets.
Section 5 summarizes this paper and outlines the research directions of future work.

2. Related Work
MV is the simplest but often effective label integration method. In MV, the label with the highest frequency is selected as
the unknown true label. However, MV naively assumes that each crowd worker has the same labeling quality, which is
almost impossible in the real world and thus weakens its performance. In recent years, label integration from multiple noisy
label sets has been extensively studied for improving its performance, and a large number of label integration methods have
been proposed from different aspects. We divide these label integration methods into two categories based on whether data
features are used.
The first category does not use data features, which only relies on multiple noisy label sets to infer unknown true labels of
instances. We first introduce the improved MV methods. For example, IWMV (Iterative Weighted Majority Voting) [14] first
uses MV’s initial integrated labels and multiple noisy label sets to estimate each crowd worker’s labeling quality. Then it
updates each crowd worker’s weight and the integrated labels of instances and iterates multiple times until convergence
or reaches the number of iterations. [25] uses the certainty information of the majority class in multiple noisy label sets
and thus proposes four soft MV methods. [28] uses multiple noisy label sets to evaluate each crowd worker’s labeling quality
by a differential evolution method. In addition to the improved MV methods, there are some other methods that do not use
data features. For example, DS (Dawid and Skene) [3] uses multiple noisy label sets to model a confusion matrix through an
expectation–maximization [26] method, where the confusion matrix represents the crowd worker’s probability of providing
each label for instances belonging to each class. GLAD (Generative model of Labels, Abilities, and Difficulties) [32] uses mul398

method, respectively. PLAT (Positive LAbel frequency Threshold) [42] uses multiple noisy label sets to model the decision

to a specific class.
The second category uses data features to help to infer unknown true labels from multiple noisy label sets. For example,
[20] proposes a fine grained truth discovery model that uses data features and multiple noisy label sets to build a probabilistic model to simultaneously estimate topic expertise and infer true labels. [23] uses the potential true label as a single hidden
layer, then uses the multiple noisy label set as the output layer, and finally uses data features to train the deep neural network (DNN) through backpropagation. BLCC (BiLayer Collaborative Clustering) [40] first generates concept-level features
from multiple noisy label sets and conducts clustering on concept-level features to infer the initially integrated labels. Then,
BLCC conducts another clustering on the physical-level features (data features) and uses the integrated labels obtained at the
conceptual layer to infer the class membership of each cluster in the physical layer. Finally, the uncertainties of instances at
the physical layer and the conceptual layer are calculated separately. For an instance with a small uncertainty at the physical
layer and a large uncertainty at the conceptual layer, the label obtained at the physical layer is assigned as its final integrated
label. [27] uses data features and multiple noisy label sets to evaluate each crowd worker’s labeling quality on different
instances and thus proposes four weighted soft MV methods. [36] introduces an active crowdsourcing learning strategy
to compute the crowd consensus, which also uses data features and multiple noisy label sets. MNLDP (Multiple Noisy Label
Distribution Propagation) [7] first uses multiple noisy label sets to calculate each instance’s multiple noisy label distribution.
Then, MNLDP uses data features to construct a feature space according to the overlapped local linear neighborhood patches
and calculates each patch’s edge weights through standard quadratic programming. Finally, each instance obtains a part of
multiple noisy label distributions from its nearest neighbors while maintaining a part of its own. According to the final label
distributions, the integrated labels of instances are obtained. [15] uses data features and multiple noisy label sets to combine
the advantages of the DNN and the probabilistic graphics model and proposes a fully bayesian deep generation crowdsourcing model (BayesDGC). BayesDGC learns each crowd worker’s confusion matrix and the distribution of potential true
labels through end-to-end training.

perform better than MV, for better application in the real world, they still need to be further improved. In this paper, we

same class as each instance. Finally, the integrated labels are inferred by weighted MV. We call our proposed label integration method as label augmented and weighted majority voting (LAWMV).

3.3. Label augmentation
In this section, we will discuss in detail how to use the KNN algorithm to find each instance’s K-nearest neighbors for
augmenting its multiple noisy label set. We first briefly introduce the traditional KNN. For a test instance, the distance
between it and each instance in a training set is first calculated. Then, the K-nearest neighbors that the distance to the test
instance is closest are found. Finally, the test instance’s class label is predicted according to class labels of its K-nearest neighbors. In order to apply it to crowdsourcing scenarios, the KNN used in our method is slightly different from the traditional
KNN. In our method, for each instance in a dataset, its K-nearest neighbors (including itself) are found in the dataset. After
this, multiple noisy label sets of each instance’s K-nearest neighbors are merged to obtain its augmented multiple noisy label
set. The calculation of the distance is crucial for finding each instance’s K-nearest neighbors.

3.5. K value setting
As we all know, the parameter K of the traditional distance-weighted KNN generally takes a small fixed value, and its performance is not sensitive to the value of K. We think this is mainly due to two points: 1) Each neighbor is weighted by distance. 2) Each neighbor’s class label (called the true label in crowdsourcing) is used to predict the test instance’s. However,
our method is different from this. Although each neighbor is also weighted, each instance uses the neighbor’s multiple noisy
labels, which are noisy and different from the true label. Therefore, for the value of K in our method, a larger value should be
taken as much as possible to reduce the influence of the noise labels in the multiple noisy label set. However, there is a problem. Assuming that the instance x1 belongs to c1 , there are only 20 instances belonging to c1 in the dataset. Obviously, setting
K to a value greater than 20 is inappropriate. In the crowdsourcing scenarios, we don’t know which class the instance belongs
to, let alone the number of instances of the class it belongs to. We only know the number of instances N and the number of
classes Q in the dataset. Therefore, we estimate the value of K by the following equation:

4. Experiments and Results
The purpose of this section is to evaluate the performance of our proposed LAWMV. Therefore, we compare LAWMV with
MV, ZC, KOS, GTIC, IWMV, and MNLDP in terms of the integration accuracy and classification accuracy. Here, the integration
accuracy represents the proportion of instances in the dataset where the integrated label is equal to the true label. The classification accuracy represents the proportion of instances in the dataset where the predicted label is equal to the true label.
In our experiments, we implemented LAWMV, IWMV, and MNLDP on the CEKA [39] platform. And in LAWMV, the parameter
b is set to 0:5. In addition, for MV, ZC, KOS and GTIC, we used the existing implementations and parameter settings on the
CEKA platform.
4.1. Experiments on Simulated Datasets
In this subsection, we conduct our experiments on the whole 34 simulated datasets from the CEKA platform. Table 1
shows the details of these datasets that represent a wide range of domains and data characteristics. Since some datasets have
missing values and some competitors cannot handle missing values. Therefore, in our experiment, we use the mean of
numerical feature values or the modes of the nominal feature values from the available data to replace all missing feature
values by the unsupervised feature filter ReplaceMissingValues in the WEKA [34] platform. After replacing all missing feature values, we simulate the crowdsourcing process to obtain the multiple noisy label sets of all instances. We first hide the
true labels of all instances. Then, we simulate crowd workers to label the instances and the probability of labeling each
instance as its original true label is p and the probability of other labels depends on their proportions in the dataset. In
our experiments, we randomly generate each crowd worker’s labeling quality p from an uniform distribution in the interval
½0:3; 0:9, and the number of crowd workers is set to 5.
After obtaining five labels of each instance, we first apply label integration methods, including MV, ZC, KOS, GTIC, IWMV,
MNLDP and our proposed LAWMV, to infer the integrated labels. Then, we use the training set with integrated labels to build
a classifier and use the test set with true labels to evaluate its performance by ten-fold cross-validation. Here, all label integration methods use C4.5 (J48) [21] as the target classifier. In order to reduce the fluctuation caused by the randomness of
the simulation process, each experiment is repeated ten times independently, and the integration accuracies and classification accuracies are averaged, respectively.

4.3. Parameter Sensitivity Analysis of LAWMV
In this paper, we use QN roughly to estimate the number of instances of each class and use b to scale QN to obtain the value of
the number of neighbors K. Therefore, the larger the b, the larger the K, and the smaller the b, the smaller the K. We believe
that as many neighbors as possible should be found for each instance in LAWMV to obtain its augmented multiple noisy label
set to achieve better performance. However, when the number of neighbors exceeds the number of instances of the class to
which the instance belongs, the performance of LAWMV decreases as K increases. In addition, the running time of LAWMV
will also be increased as K increases. Therefore, the parameter b should not be set too large nor should it be set too small.
Fig. 5 shows the integration accuracy of LAWMV when b varies from 0.1 to 1.0 on the ‘‘Leaves” dataset. We can see that
when b takes values from 0.1 to 0.4, the integration accuracy of LAWMV gradually increases as the number of neighbors
increases. When b takes a value from 0.6 to 1.0, the integration accuracy of LAWMV gradually decreases as the number of

5. Conclusions and Future Work
This paper proposes a novel label integration method, simply LAWMV. LAWMV first uses the K-nearest neighbors (KNN)
algorithm to find each instance’s K-nearest neighbors (including itself) and merges their multiple noisy label sets to obtain
its augmented multiple noisy label set. Then, the labels from different neighbors are weighted by the distances and the label
similarities between each instance and its neighbors. Finally, the integrated label of each instance is inferred by weighted
majority voting (MV). The extensive experiments on 34 simulated and two real-world crowdsourced datasets show that
LAWMV significantly outperforms all of its state-of-the-art competitors in terms of the label quality and model quality.
Our work provides a simple but effective label integration method that could be applied to a broad range of practical applications such as crowdsourced image recognition and text classification.
Seen from Eq. (9), for each instance, not only its own multiple noisy labels but also its augmented multiple noisy labels
from the same neighbor all have the same weights. Namely, the multiple noisy labels from the same instance have the same
quality, which is a little simple and rough. We believe that learning a different weight for each noisy label even from the
same instance could further improve the current LAWMV’s performance and make its advantages more stronger. This is
the main direction of our future work.
